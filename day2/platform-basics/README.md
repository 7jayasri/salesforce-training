# Day 2 - Salesforce Platform Basics

# What is Salesforce Platform?
Salesforce Platform is a cloud-based CRM and application development platform that helps businesses manage customer relationships, automate processes, and build scalable applications.
It provides:
- Data management
- Automation tools
- Security
- Reporting and dashboards
- Application development features
Salesforce follows a multi-tenant architecture where multiple organizations share the same infrastructure securely.

---

# Salesforce Platform Structure
The Salesforce platform is organized using:
- Apps
- Objects
- Tabs
- Records
- Fields
These components work together to help businesses manage data efficiently.

---

# What is an App in Salesforce?
An App in Salesforce is a collection of related tabs, objects, and functionalities designed for a specific business purpose.
### Examples:
- Sales App
- Service App
- College Management App
Apps help users access all related features from one location.

---
# What is an Object?
An Object is similar to a database table that stores data in Salesforce.
Objects contain:
- Records (Rows)
- Fields (Columns)
### Standard Objects
- Account
- Contact
- Opportunity
### Custom Objects
- Student
- Faculty
- Course

---
# What is a Tab?
A Tab is the user interface element used to access objects, records, dashboards, and applications.
### Examples:
- Accounts Tab
- Contacts Tab
- Reports Tab
Tabs make navigation easier inside Salesforce applications.

---
# CRM Concepts in Salesforce
CRM concepts are implemented using Salesforce objects.
These objects are grouped together inside apps to support business workflows.

---

# Difference Between App and Object

| App                                 |           Object               |
|--- ---------------------------------|--------------------------------|
| Collection of tools and features    | Stores business data           |
| Contains multiple objects           | Contains records and fields    |
| Used for workflow management        | Used for data management       |
| Example: Sales App                  | Example: Account Object        |

---
# Configuration vs Coding
## Configuration (No Code)
Configuration means building functionality using Salesforce tools without writing code.
### Used When:
- Requirements are simple
- Faster implementation is needed
- Maintenance should be easy
### Examples:
1. Creating custom fields
2. Building flows using Flow Builder

---
## Coding (Apex)
Coding is used when business requirements are complex and cannot be solved using configuration tools.
### Used When:
- Advanced logic is needed
- External integrations are required
- Complex automation is needed
### Examples:
1. Custom business logic using Apex
2. API integrations with external systems

---

# Multi-Tenant Architecture
Multi-tenant architecture means multiple organizations use the same Salesforce infrastructure while keeping their data secure and isolated.
### Advantages:
- Automatic updates
- Lower maintenance cost
- Better scalability
- Improved security

---

# How Developers Extend Salesforce
Salesforce allows developers to extend functionality using:
- Apex Programming Language
- Lightning Components
- APIs
- Flow Automation
- AppExchange Integrations
These tools help organizations build customized business applications.
